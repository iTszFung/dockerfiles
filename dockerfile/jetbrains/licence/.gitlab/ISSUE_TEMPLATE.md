# 问题提交模板

## 描述

问题描述：
  * 认为所存在的错误的内容
  * 提出修正建议 [option]

## 问题类型

### 按照以下的类型标注范围

* feat：新功能
* fix：修复bug
* doc：文档改变
* style：代码格式改变
* refactor：某个已有功能重构
* perf：性能优化
* test：增加测试
* build：改变了build工具 如 grunt换成了 npm
* revert：撤销上一次的 commit

### 标记作用域

* $compile：编译时出现的问题
* $location：运行时某些函数出现的一些问题
* $browser：在浏览器运行的时候就出现的问题
* $resource：对应资源或资源目录出现的问题
* $\*：影响的是整个项目的问题
